需求

今年 DOCKER宣布支持 K8S
所以就玩看看囉 (記得抓EDGE版本才有K8S)

指令步驟

建置WEBAPI並且測試

寫Dockerfile + Build + RUN Test

在終端機做測試

寫K8S file + DEPLOY

用VSCode記得安裝外掛 kubernetes Support + Kubernetes
創一個 deploy.yaml檔案
在畫面中打dn按Tab就會自帶snippets 版本如下

我的版本，應該可以更精簡，後面再說每個標籤是在幹嘛的

再跑kubectl veriosn發現了一個很異常的問題。
會一直出現連不到server的問題
找了很久
發現是我使用Kubectl出現相衝問題 (因為電腦有灌 minikube , Google CloudSDK)
所以設定跑到GOOGLE的K8S那邊的Server
解法如下：
通過指令先移除 GOOGLE k8S

再透過DOCKER 介面做SETTING RESET KUBERNETES設定
再透過指令切換到DOCKER的K8S即可

這是轉換到DOCKER K8s的指令

接著就開始Deploy

檢查是否都RUN成功

然後找到EndPort 接著輸入 http://127.0.0.1:EndPort/api/Values
就OK囉~
